## **E5 Integration: Context Size Issue & Fix (00:41 UTC)**

- **Error in logs**: "Input is longer than the context size. Try to increase the context size or use another model that supports longer contexts."
- **Root cause**: E5-small-v2 has **512 token context window**
- **OpenClaw default**: 400 token chunks + 80 overlap = 480 tokens
- **Problem**: Too close to limit, needs safety buffer

### **Configuration Fix Applied:**
```json5
"memorySearch": {

  "chunkOverlap": 64       // Reduced from 80
}
```

#### **Technical Specifications:**

- **Problem**: 480/512 = 94% of capacity (insufficient buffer)
- **Safety buffer needed**: Typically 10-20% (50-100 tokens)
- **Result**: "Input is longer than the context size" error
