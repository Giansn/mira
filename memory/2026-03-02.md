### Entrosana.com Dashboard Connection Crisis & Resolution
- **22:05 UTC**: Dashboard inaccessible via entrosana.com - "disconnected (1006): no reason"
- **22:09 UTC**: Began systematic troubleshooting of 3-layer firewall architecture

#### **Firewall Layer Analysis:**
1. **AWS Security Group**: Port 443 allowed `0.0.0.0/0` ✅
2. **AWS Network ACL**: Critical issues found ❌
3. **Instance Firewall (UFW)**: Critical issues found ❌

#### **Network ACL Issues Discovered:**
- **NACL ID**: `acl-09f40664fa1e8bc26` (associated with subnet `subnet-0b6c63389dc048953`)
- **Problem 1**: INBOUND Rule 101 only allowed port 443, blocking proxied port 18789
- **Problem 2**: OUTBOUND rules didn't allow ephemeral ports (1024-65535) for responses
- **Root cause**: NACLs are stateless - need separate inbound AND outbound rules

#### **UFW Issues Discovered:**
- **Status**: UFW active with default `deny (incoming)`
- **Problem**: Port 443 not in allow list, only ports 22 and 18789 allowed
- **Command**: `sudo ufw status verbose` revealed the blockage

#### **Step-by-Step Resolution:**
1. **22:12 UTC**: Added ephemeral port outbound rule (1024-65535) to NACL
2. **22:28 UTC**: Changed INBOUND Rule 101 from "HTTPS (443)" to "All Traffic"
3. **23:08 UTC**: Added UFW rule: `sudo ufw allow 443/tcp`

#### **Final NACL Configuration:**
**INBOUND Rules:**
```
100: ALLOW All Traffic (0.0.0.0/0)
101: ALLOW All Traffic (0.0.0.0/0)  # Changed from port 443 only
*: DENY All Traffic (0.0.0.0/0)
```

**OUTBOUND Rules:**
```
100: ALLOW All Traffic (0.0.0.0/0)
102: ALLOW TCP 1024-65535 (0.0.0.0/0)  # Added for ephemeral ports
*: DENY All Traffic (0.0.0.0/0)
```

#### **Final UFW Configuration:**
```
Port 22: ALLOW IN Anywhere
Port 443: ALLOW IN Anywhere  # Added
Port 18789: ALLOW IN Anywhere
Default: deny (incoming)
```
